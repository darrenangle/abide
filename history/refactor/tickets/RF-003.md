# RF-003: Align Sonnet-Family Claims With Actual Verification

- Priority: `P1`
- Status: `done`
- Area: `form fidelity`
- Origin: `AUDIT-002`

## Problem

The sonnet classes describe iambic pentameter but currently verify syllable counts, not meter.

## Primary Files

- `src/abide/forms/sonnet.py`
- related sonnet variants in `src/abide/forms/extended_sonnet.py`

## Acceptance Criteria

- Either enforce actual meter where claims say meter, or rewrite descriptions/instructions to clearly say syllable proxy.
- Preserve usable partial-credit reward behavior.
- Add tests proving the description matches what is enforced.

## Resolution

- Rewrote sonnet-family public descriptions to say they verify approximately 10 syllables per line rather than full iambic meter.
- Updated related sonnet-variant descriptions and prompt-spec text to use the same syllable-proxy language.
- Added tests that lock the description/spec wording to the verified syllable behavior.
