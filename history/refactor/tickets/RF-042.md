## Problem

Several short-form families now reject extra or missing lines correctly, but that behavior still lives in one-off probes or narrow form-specific tests. We need a shared mutation harness so the repo keeps enforcing those exact-count guarantees across Japanese, shape, Fibonacci, and related short forms.

## Primary Files

- `tests/forms/`

## Acceptance Criteria

- Add a shared mutation harness covering representative exact-count short forms with stable valid examples.
- Assert that each baseline example passes canonically.
- Assert that both a trailing added line and a dropped final line fail canonically for every covered form.
