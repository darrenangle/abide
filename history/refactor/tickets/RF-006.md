# RF-006: Use Exact Form Metadata In TRL Reward Path

- Priority: `P1`
- Status: `todo`
- Area: `training integration`
- Origin: `AUDIT-004`

## Problem

The TRL reward path currently guesses form identity from prompt substrings, which can score completions against the wrong form.

## Primary Files

- `scripts/train_grpo_trl.py`

## Acceptance Criteria

- TRL dataset preserves structured `form_name`.
- Reward function reads exact metadata instead of parsing prompt text.
- Add a regression test for overlapping names such as `Ode` and `PindaricOde`.
