# RF-058: Eliminate High-Score Empty-Input Reward Leaks In Remaining Public Forms And Constraints

- Priority: `P1`
- Status: `done`
- Area: `reward integrity`
- Origin: `MANUAL-VERIFIER-AUDIT-2026-04-01`

## Problem

Several exported forms still assign very high scores to an empty poem even
while returning `passed=False`, which creates a reward leak for RL training.
Reproduced cases from the current default catalog include `ElementalVerse`
(`0.904`), `ExclamationEcho` (`0.904`), `OddEvenDance` (`0.901`),
`QuestionQuest` (`0.914`), `TemporalVerse` (`0.901`), `ThunderVerse`
(`0.914`), and `WhisperPoem` (`0.901`). The public `EndRhymeDensity`
constraint also returns `passed=True, score=1.0` on empty input.

## Primary Files

- `src/abide/forms/novel.py`
- `src/abide/constraints/relational.py`
- `tests/forms/test_exported_form_surface.py`
- `tests/forms/test_novel_forms.py`
- `tests/constraints/test_relational.py`

## Acceptance Criteria

- Empty poem input must not receive near-perfect scores from the remaining
  exported forms or the public `EndRhymeDensity` constraint.
- Add direct regressions for the reproduced high-score empty-input cases.
- Keep non-empty near-miss behavior intact where possible; this ticket is about
  blocking empty-input reward leaks, not flattening all partial credit.

## Completed

- Added an empty-safe steep-score helper for the affected novel forms so empty
  input now receives `0.0` instead of near-perfect partial credit.
- Patched `EndRhymeDensity` so empty poems fail instead of passing vacuously,
  while preserving the single-line behavior for non-empty edge cases.
- Added direct regressions for the reproduced empty-input reward leaks in the
  novel-form family and the public rhyme-density constraint.
