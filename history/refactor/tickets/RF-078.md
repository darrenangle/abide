# RF-078: Update Verifiers To The Current RL Stack And Rewrite The Gemma Verifiers Trainer Around Well-Known Forms

- Priority: `P2`
- Status: `done`
- Area: `training reliability`
- Origin: `VERIFIERS-REFRESH-2026-04-03`

## Problem

The repo’s verifiers-based trainer is behind the current upstream verifiers
stack. The lockfile still pins `verifiers==0.1.10`, the training extra no
longer installs the RL package that `verifiers.rl.trainer` now expects, and
the current `train_grpo.py` still reflects older broad-form and ad hoc runner
patterns. We need to refresh the dependency wiring against the latest official
guidance and rewrite the Gemma verifiers training path around a focused subset
of well-known forms.

## Primary Files

- `pyproject.toml`
- `uv.lock`
- `scripts/train_grpo.py`
- `scripts/run_grpo*.sh`
- `scripts/prompt_generator.py`
- `src/abide/forms/catalog.py`
- `tests/integration/`

## Acceptance Criteria

- The repo must update to the current stable `verifiers` release and install
  the RL trainer stack correctly.
- The rewritten verifiers trainer must target a curated subset of well-known
  forms rather than the previous broad/open-ended modes by default.
- The training path should align with current verifiers best practices where
  they apply to a local single-node script: clear `SingleTurnEnv` usage,
  metadata-driven rewards, parser/metric support where useful, and explicit
  acknowledgement that `vf.RLTrainer` is legacy/small-scale.
- The new dependency and trainer behavior must be covered by tests and direct
  local verification.

## Outcome

- Updated the repo to `verifiers>=0.1.11` in the main env and refreshed
  `uv.lock`.
- Rewrote `scripts/train_grpo.py` around a smaller well-known-form subset,
  parser-aware reward plumbing, and the current legacy-verifiers guidance.
- Added `scripts/prepare_verifiers_runtime.sh` plus updated Gemma runners so
  the split `verifiers-rl` stack is installed into `.venv-verifiers` instead of
  conflicting with the repo's TRL runtime.
- Added regression coverage for the new catalog subset, prompt-selection mode,
  trainer CLI/config behavior, and the isolated legacy-runtime script.
