# RF-033: Remove The `pronouncing` Dependency And Read CMU Data Directly From `cmudict`

- Priority: `P1`
- Status: `done`
- Area: `phonetics dependency chain`
- Origin: `FOLLOWUP-032`

## Problem

The repo currently depends on `pronouncing`, which imports `pkg_resources` and drags `setuptools` into the runtime path. We only use one `pronouncing` helper for CMU dictionary lookup, and `cmudict` already provides the raw data directly.

## Primary Files

- `src/abide/primitives/phonetics.py`
- `pyproject.toml`
- `uv.lock`

## Acceptance Criteria

- `phonetics.py` must fetch phoneme variants directly from `cmudict`.
- Remove `pronouncing` and the extra `setuptools` runtime dependency from project dependencies.
- Remove the temporary warning-suppression workaround added for the old import path.
- `uv run pytest -q -W error` must still pass cleanly.

## Completed

- Switched `get_phonemes()` to read directly from `cmudict.dict()` and removed the `pronouncing` import path entirely.
- Removed the explicit `pronouncing` and `setuptools` runtime dependency entries from `pyproject.toml` and refreshed `uv.lock`.
- Verified with `uv run ruff check src/abide tests scripts`, `uv run mypy src/abide`, and `uv run pytest -q -W error` -> `611 passed, 1 skipped`.
