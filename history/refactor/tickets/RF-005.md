# RF-005: Reject Dangling Lines In Ghazal Verification

- Priority: `P1`
- Status: `done`
- Area: `form verifier correctness`
- Origin: `AUDIT-003`

## Problem

`Ghazal.verify()` computes `num_couplets` with floor division and silently ignores an unmatched extra line.

## Primary Files

- `src/abide/forms/ghazal.py`
- `tests/fixtures/test_form_validation.py`

## Acceptance Criteria

- Odd line counts fail or receive a severe penalty.
- The verifier does not silently drop unmatched lines.
- Add a regression test for a valid ghazal plus one extra line.

## Resolution

- Added a `Complete couplets` rubric item keyed off even nonblank line count.
- Odd line counts now sharply reduce score and force `passed=False`.
- Added regression coverage for the dangling-line case in `tests/fixtures/test_form_validation.py`.
