# RF-057: Harden Remaining Character And Sound Lexical Constraint Constructors

- Priority: `P1`
- Status: `done`
- Area: `constraint contract validation`
- Origin: `MANUAL-VERIFIER-AUDIT-2026-04-01`

## Problem

The remaining character- and sound-oriented lexical constraints still allow a
few zero-bound or invalid-position configurations. Reproduced cases include
`CharacterCount([])`, `CharacterCount(0)`, `Alliteration(letter=\"S\", min_words=0)`,
`Alliteration(min_consecutive=0)`, `MonosyllabicOnly(min_words=0)`, and
`CharacterPalindrome(lines=[0])`.

## Primary Files

- `src/abide/constraints/lexical.py`
- `tests/constraints/test_lexical.py`

## Acceptance Criteria

- Character- and sound-oriented lexical constraints must reject empty counts,
  zero minima, negative tolerances, and invalid 1-based line-number lists
  instead of crashing later or passing vacuously.
- Add direct regressions for the reproduced degenerate constructor cases.

## Completed

- Hardened the remaining character- and sound-oriented lexical primitives so they now reject empty count patterns, zero minima, negative tolerances, and invalid 1-based line-number lists instead of crashing or passing vacuously.
- Added constructor validation to `CharacterCount`, `Alliteration`, `MonosyllabicOnly`, and `CharacterPalindrome`, covering the reproduced zero-count, zero-minimum, and invalid line-index cases.
- Added direct regressions in `test_lexical.py` and verified with `uv run ruff check src/abide tests scripts`, `uv run mypy src/abide`, and `uv run pytest -q -W error` -> `803 passed, 1 skipped`.
