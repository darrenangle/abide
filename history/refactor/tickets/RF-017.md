# RF-017: Add Constrained-Form Coverage And Fix Catalog Instantiation

- Priority: `P1`
- Status: `done`
- Area: `semantic constraint forms`
- Origin: `FOLLOWUP-009`

## Problem

The mechanically checkable forms in `constrained.py` still have no direct behavioral coverage, and the shared form catalog currently passes the wrong constructor kwargs to at least `Mesostic` and `Anaphora`. That means generic form loading can silently skip or break exported forms that should be straightforward to verify reliably.

## Primary Files

- `src/abide/forms/constrained.py`
- `src/abide/forms/catalog.py`
- `tests/forms/`

## Acceptance Criteria

- Add direct behavioral coverage for `Abecedarian`, `Lipogram`, `Univocalic`, `Mesostic`, `Anaphora`, and `PalindromePoem`.
- Fix any catalog/default-instantiation bugs for these forms.
- Confirm the public descriptions and constructor usage match the code that actually runs.

## Notes

- Completed:
  - added direct behavioral coverage for `Abecedarian`, `Lipogram`, `Univocalic`, `Mesostic`, `Anaphora`, and `PalindromePoem`
  - fixed `SPECIAL_FORM_KWARGS` so `Mesostic` and `Anaphora` instantiate correctly through the shared catalog
  - improved `Anaphora` auto-detection to prefer the longest repeated opening rather than collapsing to a single repeated token
  - made `Abecedarian` ignore leading punctuation so quoted or parenthesized line openings do not fail spuriously
