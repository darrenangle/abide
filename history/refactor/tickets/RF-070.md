# RF-070: Add Training-Time Reward Telemetry And Align Stale Baguettotron Reward Routing

- Priority: `P2`
- Status: `done`
- Area: `training reliability`
- Origin: `TRAINING-CANARY-PREP-2026-04-02`

## Problem

The repo is now in a good enough state for canary RL runs, but the current
training scripts still make debugging harder than it needs to be. They do not
aggregate per-form reward telemetry, zero-reward rates, or failure-reason
summaries during training. Separately, the Baguettotron-specific TRL script is
still on the older prompt path and does not preserve exact `form_name`
metadata through its dataset/reward route.

## Primary Files

- `scripts/train_grpo.py`
- `scripts/train_grpo_trl.py`
- `scripts/train_grpo_baguettotron.py`
- `scripts/reward_telemetry.py`
- `tests/integration/`

## Acceptance Criteria

- The main training scripts must emit periodic aggregate reward telemetry,
  including per-form mean reward, pass rate, and zero-reward rate.
- Telemetry must include a compact summary of common failure reasons.
- The stale Baguettotron TRL script must use exact `form_name` metadata instead
  of prompt substring guessing.
- The new helper behavior must be covered by integration tests.

## Resolution

- Added `scripts/reward_telemetry.py` to aggregate per-form mean reward, pass
  rate, zero-rate, and compact failure-reason summaries with optional Weights &
  Biases logging.
- Wired periodic telemetry emission and end-of-run flushing into
  `train_grpo.py`, `train_grpo_trl.py`, and `train_grpo_baguettotron.py`.
- Moved the Baguettotron TRL path onto exact `form_name` metadata in both
  dataset creation and reward routing, removing prompt substring guessing.
- Added integration coverage for telemetry aggregation and the Baguettotron
  metadata path.
