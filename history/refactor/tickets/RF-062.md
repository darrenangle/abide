# RF-062: Reject Vacuous Comparison Passes And Punctuation-Only Reward Leaks

- Priority: `P1`
- Status: `done`
- Area: `reward integrity`
- Origin: `DEGENERATE-INPUT-AUDIT-2026-04-01`

## Problem

Some comparison-based primitives and forms were still treating missing evidence
as success. Reproduced cases included `NoSharedLetters("consecutive")` on
punctuation-only input returning `passed=True, score=1.0`,
`AlternatingIsolation("!!!\\n???\\n...\\n!!!\\n???")` scoring `0.882`,
`EchoEnd("alpha")` scoring `0.914`, `ConsonantCascade("alpha\\nbravo")`
scoring `0.901`, `ConsonantCascade` on punctuation-only input scoring `0.932`,
and `SandwichSonnet` on punctuation-only input scoring `0.932`.

## Primary Files

- `src/abide/constraints/lexical.py`
- `src/abide/forms/novel.py`
- `tests/constraints/test_lexical.py`
- `tests/forms/test_novel_forms.py`
- `tests/forms/test_hard_forms.py`

## Acceptance Criteria

- `NoSharedLetters` must fail when a compared line pair lacks alphabetic
  content instead of treating that pair as a success.
- Comparison-based novel forms must no longer award near-perfect scores to
  punctuation-only or too-small samples.
- Existing valid comparison examples should remain valid.

## Completed

- Patched `NoSharedLetters` so pairs without alphabetic content fail instead of
  passing vacuously.
- Patched `EchoEnd`, `ConsonantCascade`, and `SandwichSonnet` so they require
  meaningful comparison samples and penalize missing required comparisons.
- Added direct regressions for the reproduced primitive, novel-form, and
  hard-form vacuity cases.
