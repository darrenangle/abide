# RF-093: Harden Abecedarian Against Single-Letter Leading Token Exploits

- Priority: `P1`
- Status: `done`
- Area: `verifier correctness`
- Origin: `user request`
- Depends On: `RF-090`

## Problem

`Abecedarian` currently accepts lines whose first alphabetic character matches
the target letter even when the leading token is not a meaningful word. That
lets junk lines like `B quiet field rests` pass, which is structurally legal
under the current implementation but not a useful poetic line.

## Primary Files

- `src/abide/forms/constrained.py`
- `tests/forms/test_constrained_forms.py`

## Acceptance Criteria

- `Abecedarian` should validate the first word, not only the first alphabetic
  character anywhere in the line.
- Single-letter leading tokens should be rejected except for valid one-letter
  words like `A` and `I`.
- Existing punctuation-tolerant behavior should remain intact.
- Add a regression covering `B quiet field rests`-style junk openings.

## Result

- `Abecedarian` now validates the first alphabetic word on each line instead of
  only the first alphabetic character anywhere in the line.
- Single-letter leading tokens are rejected unless they are valid one-letter
  words (`A` or `I`).
- Punctuation-tolerant openings still pass.
- Added a regression for `B quiet field rests`.
