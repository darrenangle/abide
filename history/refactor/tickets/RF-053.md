# RF-053: Reject Degenerate Shell-Form Bounds And Empty Generic-Form Constructors

- Priority: `P1`
- Status: `done`
- Area: `constructor contract validation`
- Origin: `MANUAL-VERIFIER-AUDIT-2026-04-01`

## Problem

Several generic shell forms still accept zero-sized or inverted constructor
bounds and either pass empty input outright or construct internally
contradictory verifiers. Reproduced cases include `Ode(min_lines=0)`,
`IrregularOde(min_lines=0, min_stanzas=0)`, `PindaricOde(triads=0)`,
`Aubade(stanza_count=0)`, `FreeVerse(min_lines=0, min_stanzas=0)`,
`ProsePoem(min_paragraphs=0, max_paragraphs=0, min_sentences=0)`, and
`BlankVerse(min_lines=0)`.

## Primary Files

- `src/abide/forms/ode.py`
- `src/abide/forms/free_verse.py`
- `src/abide/forms/blank_verse.py`
- `src/abide/forms/modern.py`
- `tests/forms/test_generic_form_claims.py`
- `tests/forms/test_fibonacci_and_free_forms.py`
- `tests/forms/test_world_and_modern_forms.py`

## Acceptance Criteria

- Generic shell forms must reject nonpositive required size bounds instead of
  constructing vacuous or contradictory verifiers.
- Constructors with paired lower/upper bounds must reject inverted ranges.
- Stanzaic shells must reject zero-sized stanza or triad counts and other
  silently truncated layout parameters.
- Add direct regression tests for the reproduced degenerate constructor cases.

## Completed

- Added shared constructor-bound validation helpers for shell-form configuration and used them to reject zero-sized, inverted, and silently truncated shell-form parameters early.
- Hardened `Ode`, `HoratianOde`, `PindaricOde`, `IrregularOde`, `Aubade`, `FreeVerse`, `ProsePoem`, `BlankVerse`, `DramaticVerse`, and `Skeltonic` against degenerate constructor values.
- Added direct constructor-contract regressions covering the reproduced empty-pass cases plus inconsistent stanza/rhyme layout parameters.
- Verified with `uv run ruff check src/abide tests scripts`, `uv run mypy src/abide`, and `uv run pytest -q -W error` -> `716 passed, 1 skipped`.
