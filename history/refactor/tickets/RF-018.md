# RF-018: Harden Mathematical Forms Against Prefix-Pass And Token-Count False Positives

- Priority: `P1`
- Status: `done`
- Area: `mathematical forms`
- Origin: `FOLLOWUP-010`

## Problem

Several mathematically structured forms currently validate only the expected prefix of the poem and ignore extra trailing lines or stanzas. That lets malformed outputs report `passed=True` as long as the leading segment satisfies the form. `SelfReferential` also counts number words by substring, so fragments can be treated as valid number tokens.

## Primary Files

- `src/abide/forms/mathematical.py`
- `tests/forms/`

## Acceptance Criteria

- `GoldenRatioVerse`, `PythagoreanTercet`, `CoprimeVerse`, `SquareStanzas`, and `SelfReferential` must fail canonically when extra trailing lines or stanzas are present.
- `SelfReferential` must count whole number tokens rather than raw substrings.
- Add direct regression coverage for the reproduced false positives and for valid baseline examples.
- Keep descriptions aligned with the behavior actually enforced.

## Notes

- Completed:

  - reproduced and fixed the trailing-line / trailing-stanza false positives across those forms
  - changed `SelfReferential` to count whole number tokens via token boundaries rather than raw substrings
