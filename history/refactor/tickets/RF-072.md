# RF-072: Make `uv sync --extra training` Reproducible For Flash-Attn Build Isolation

- Priority: `P2`
- Status: `done`
- Area: `training environment`
- Origin: `GEMMA4-CANARY-2026-04-02`

## Problem

The new Gemma 4 E4B canary path is committed, but the local training
environment still does not sync cleanly. `uv sync --extra training` fails while
building `flash-attn` because `torch` is missing from the isolated build
environment. That means the repo still depends on manual installation steps
instead of having a reproducible training extra.

## Primary Files

- `pyproject.toml`
- `tests/integration/`

## Acceptance Criteria

- `pyproject.toml` must include the uv build-isolation configuration needed for
  `flash-attn` under the training extra.
- The configuration must be covered by a regression test.
- `uv sync --extra training` must get past the previous missing-`torch`
  build-isolation failure.

## Resolution

- Added uv build-isolation configuration for `flash-attn` in
  `pyproject.toml`, matching `torch` at build time and opting into the
  prebuilt-wheel path.
- Added a regression test in
  `tests/integration/test_pyproject_training_sync.py`.
- Verified that `uv sync --extra training` now gets past the previous
  missing-`torch` build-isolation failure and completes successfully.
