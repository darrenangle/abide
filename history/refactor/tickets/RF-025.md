# RF-025: Narrow Generic-form Claims To The Structure The Code Actually Verifies

- Priority: `P2`
- Status: `done`
- Area: `public form descriptions and docs`
- Origin: `FOLLOWUP-017`

## Problem

Some generic forms still advertise thematic or literary properties that the verifier does not detect. For example, `Ode()` will accept any poem in the configured line-count range, yet the surrounding docs still call it a lyric poem addressed to a subject. `Aubade` and `Epigram` similarly describe thematic or tonal qualities that are not verified. That weakens the repo’s claim reliability even when the underlying structural checks behave correctly.

## Primary Files

- `src/abide/forms/ode.py`
- `src/abide/forms/modern.py`
- `src/abide/forms/epigram.py`
- `src/abide/forms/__init__.py`
- `README.md`
- `tests/forms/`

## Acceptance Criteria

- Generic/semantic form descriptions and docstrings must only promise the structure or proxy checks the code actually enforces.
- Add direct tests pinning the narrowed descriptions for representative generic forms.
- Keep all repo verification gates green after the wording cleanup.
