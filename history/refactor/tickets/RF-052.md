# RF-052: Reject Empty Explicit Target Strings In Anaphora And EchoEnd

- Priority: `P1`
- Status: `done`
- Area: `constructor contract validation`
- Origin: `MANUAL-VERIFIER-AUDIT-2026-04-01`

## Problem

`Anaphora(phrase=\"\")` and `EchoEnd(letter=\"\")` currently treat an explicitly empty target as if the caller requested auto-detection. That silently changes the verifier mode instead of rejecting an invalid explicit target.

## Primary Files

- `src/abide/forms/constrained.py`
- `src/abide/forms/novel.py`
- `tests/forms/test_constrained_forms.py`
- `tests/forms/test_novel_forms.py`

## Acceptance Criteria

- `Anaphora` must reject empty explicit phrases instead of falling back to auto-detect.
- `EchoEnd` must reject empty or non-single-character explicit letters instead of falling back to auto-detect.
- Add direct regressions for the reproduced silent-fallback cases.

## Completed

- Added explicit target validation to `Anaphora` and `EchoEnd` so empty strings and invalid letters fail fast instead of silently toggling the verifier into auto-detect mode.
- Added direct regressions for the reproduced silent-fallback cases.
- Verified with `uv run ruff check src/abide tests scripts`, `uv run mypy src/abide`, and `uv run pytest -q -W error` -> `688 passed, 1 skipped`.
