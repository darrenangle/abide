# RF-041: Reject Flattened Single-Block Terza Rima Layouts Instead Of Chunking Them Back Into Tercets

- Priority: `P1`
- Status: `done`
- Area: `terza rima stanza semantics`
- Origin: `MUTATION-SWEEP-2026-04-01`

## Problem

`TerzaRima.verify()` currently passes a single-block poem with no stanza breaks because it asks `GroupedStanzas` to chunk the lines back into tercets synthetically. That contradicts the form's own stated interlocking-tercet structure.

## Primary Files

- `src/abide/forms/terza_rima.py`
- `tests/forms/`

## Acceptance Criteria

- `TerzaRima` must not report `passed=True` for flattened single-block layouts.
- Keep the canonical multi-tercet fixture passing.
- Add a direct regression for the reproduced single-block false positive.

## Completed

- Disabled synthetic single-block stanza chunking in `TerzaRima` so the form now requires explicit tercet layout.
- Added the terza-rima fixture to the shared stanza-flattening mutation harness.
- Verified with `uv run ruff check src/abide tests scripts`, `uv run mypy src/abide`, and `uv run pytest -q -W error` -> `637 passed, 1 skipped`.
