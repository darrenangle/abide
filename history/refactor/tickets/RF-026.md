# RF-026: Strip Remaining Unverified Thematic Claims From Form Docstrings And Catalog Summaries

- Priority: `P2`
- Status: `done`
- Area: `source docs and catalog summaries`

## Problem

After RF-025, several form modules and class docstrings still describe historical themes or literary effects that the code does not verify. Examples include humor/biography for `Clerihew`, narrative content for `Ballad`, love/nature/social-commentary framing for `Tanaga`, and thematic self-containment for `Ghazal`. Even when `describe()` is already structural, these docstrings still create a misleading picture of what the verifier proves.

## Acceptance Criteria

- Remove or qualify remaining unverified thematic/literary claims in representative form docstrings and top-level catalog summaries.
- Keep structural/historical context only where it is clearly identified as non-verified background.
- Add direct tests for representative docstrings so the source-level claim drift does not come back.
- Keep all repo verification gates green.
