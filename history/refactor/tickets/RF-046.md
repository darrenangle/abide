# RF-046: Reject Unsupported PrimeVerse Line Counts Instead Of Cycling The Prime Pattern

- Priority: `P1`
- Status: `done`
- Area: `novel form constructor validation`
- Origin: `MANUAL-VERIFIER-AUDIT-2026-04-01`

## Problem

`PrimeVerse(num_lines=7)` currently passes canonically by cycling its 6-prime word-count pattern through `WordCount`, so line 7 reuses the 2-word expectation from line 1. That undocumented wraparound is not part of the form's stated contract.

## Primary Files

- `src/abide/forms/novel.py`
- `tests/forms/test_novel_forms.py`

## Acceptance Criteria

- `PrimeVerse` must reject unsupported line counts instead of silently wrapping the prime pattern.
- Supported configurations must keep working.
- Add a regression for the reproduced `num_lines=7` wraparound bug.

## Completed

- Added constructor validation to `PrimeVerse` so unsupported line counts fail fast instead of wrapping the prime sequence into an undocumented cyclic pattern.
- Added a direct regression for the reproduced `num_lines=7` wraparound bug.
- Verified with `uv run ruff check src/abide tests scripts`, `uv run mypy src/abide`, and `uv run pytest -q -W error` -> `665 passed, 1 skipped`.
