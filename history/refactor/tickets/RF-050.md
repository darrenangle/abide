# RF-050: Reject Empty Or Non-Alphabetic Target Sequences In Acrostic-Like Forms

- Priority: `P1`
- Status: `done`
- Area: `constructor contract validation`
- Origin: `MANUAL-VERIFIER-AUDIT-2026-04-01`

## Problem

Several target-sequence forms accept empty strings and become degenerate zero-length verifiers that can pass an empty poem with `score=1.0`. Reproductions: `Abecedarian(letters=\"\")`, `Mesostic(word=\"\")`, and `AlphabeticTerminus(letters=\"\")`. `DescendingStaircase(start_words=0)` also constructs a nonsensical 0-line form instead of rejecting the configuration.

## Primary Files

- `src/abide/forms/constrained.py`
- `src/abide/forms/novel.py`
- `tests/forms/test_constrained_forms.py`
- `tests/forms/test_novel_forms.py`

## Acceptance Criteria

- `Abecedarian`, `Mesostic`, and `AlphabeticTerminus` must reject empty or non-alphabetic target sequences.
- `DescendingStaircase` must reject nonpositive `start_words`.
- Add direct regressions for the reproduced degenerate-constructor cases.

## Completed

- Added explicit validation to `Abecedarian`, `Mesostic`, `AlphabeticTerminus`, and `DescendingStaircase` so degenerate empty targets and nonpositive staircase sizes fail fast instead of creating meaningless verifiers.
- Added direct regressions for the reproduced degenerate-constructor cases.
- Verified with `uv run ruff check src/abide tests scripts`, `uv run mypy src/abide`, and `uv run pytest -q -W error` -> `672 passed, 1 skipped`.
