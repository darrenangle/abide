# RF-049: Validate PalindromePoem Levels And ModularVerse Moduli Explicitly

- Priority: `P1`
- Status: `done`
- Area: `constructor contract validation`
- Origin: `MANUAL-VERIFIER-AUDIT-2026-04-01`

## Problem

`PalindromePoem(level=\"bogus\")` currently falls back silently to word-level behavior, and `ModularVerse(n=0)` raises a raw `ZeroDivisionError` from `%` normalization instead of reporting an invalid configuration clearly.

## Primary Files

- `src/abide/forms/constrained.py`
- `src/abide/forms/mathematical.py`
- `tests/forms/test_constrained_forms.py`
- `tests/forms/test_mathematical_forms.py`

## Acceptance Criteria

- `PalindromePoem` must reject unsupported `level` values instead of silently reinterpreting them.
- `ModularVerse` must validate `n > 0` and fail with `ValueError` instead of crashing with `ZeroDivisionError`.
- Add direct regressions for both reproduced invalid-configuration cases.

## Completed

- Added explicit validation to `PalindromePoem` and `ModularVerse` so unsupported `level` values and nonpositive moduli fail with clear `ValueError`s instead of silently falling back or crashing indirectly.
- Added direct regressions for both invalid-configuration cases.
- Verified with `uv run ruff check src/abide tests scripts`, `uv run mypy src/abide`, and `uv run pytest -q -W error` -> `670 passed, 1 skipped`.
