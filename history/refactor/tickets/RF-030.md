# RF-030: Gate `LiteraryBallad` Canonical Passes On Stanza Count Instead Of Blended Score Alone

- Priority: `P1`
- Status: `done`
- Area: `ballad-family pass semantics`
- Origin: `SWEEP-2026-04-01`

## Problem

`LiteraryBallad` currently uses a lenient weighted threshold with no required children. A poem with enough total lines but only a single stanza can therefore pass canonically even though stanza count is a defining part of the form's stated contract.

## Primary Files

- `src/abide/forms/ballad.py`
- `tests/forms/`

## Acceptance Criteria

- `LiteraryBallad` must not report `passed=True` unless both minimum line count and minimum stanza count pass.
- Keep partial credit behavior for near misses so reward shaping still works.
- Add a regression for the reproduced single-stanza false positive and a valid contrasting example.

## Completed

- Added required-child gating to the lenient `WeightedSum` so both line count and stanza count must pass for `passed=True`.
- Added regressions for the reproduced single-stanza false positive and a valid multi-stanza example.
- Verified with `uv run ruff check src/abide tests scripts`, `uv run mypy src/abide`, and `uv run pytest` -> `609 passed, 1 skipped`.
