# RF-092: Add a Small-Model Poemness Judge Layer for SFT Corpus Filtering

- Priority: `P1`
- Status: `done`
- Area: `training data quality`
- Origin: `user request`
- Depends On: `RF-090`, `RF-091`

## Problem

The deterministic form verifiers prove structural compliance, but the full-surface
audit showed they are still gameable by low-effort shells that satisfy the letter
of the contract without behaving like useful poem examples.

We need a second gate that judges whether a structurally valid output is still a
real poem-like warmup example rather than placeholder text, repeated scaffolding,
or obvious constraint gaming.

## Primary Files

- `src/abide/training/codex_spark_warmup.py`
- `data/sft/codex_spark_full_surface_quality_validated.summary.json`
- `history/refactor/`

## Acceptance Criteria

- Define a narrow poemness/anti-gaming rubric for a small-model judge.
- Add a durable JSONL task/result format for model-judge passes.
- Use the smallest available subagent model in this environment as the first
  judge implementation.
- Record explicit disagreement cases where structure passes but the judge rejects.
- Keep the structural verifier as a separate required gate.

## Result

- Added a durable judge-task and merged-judgment flow in
  `src/abide/training/poemness_judge.py`,
  `scripts/build_poemness_judge_tasks.py`, and
  `scripts/merge_poemness_judge_results.py`.
- Used `gpt-5.4-mini` subagents as the first cheap judge implementation on a
  `58`-item audit batch: `38` heuristic rejects plus `20` accepted controls.
- Recorded merged judgments in:
  - `data/sft/codex_spark_full_surface_judge_tasks.jsonl`
  - `data/sft/codex_spark_full_surface_judgments.jsonl`
  - `data/sft/codex_spark_full_surface_judgments.summary.json`
- The judge rejected `45 / 58` rows overall.
- Disagreement set:
  - `12` accepted controls were rejected by the judge
  - `5` heuristic rejects were accepted by the judge

## Decision

The judge adds real signal instead of merely mirroring the heuristic filter. It
should stay as a second gate beside deterministic verification and heuristic
checks, not replace them.
