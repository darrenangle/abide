# RF-010: Add A Training-Safe Form Catalog And Adversarial Harness

- Priority: `P1`
- Status: `done`
- Area: `training reliability`
- Origin: `FOLLOWUP-002`

## Problem

The codebase is now much cleaner, but training still treats too many forms as equally trustworthy. Prompt generation and reward lookup default to broad form sets, and the repo still lacks a generated adversarial harness for the subset we actually want to trust in RL runs.

## Primary Files

- `src/abide/forms/catalog.py`
- `scripts/prompt_generator.py`
- `scripts/train_grpo.py`
- `scripts/train_grpo_trl.py`
- `tests/forms/test_catalog.py`
- `tests/forms/test_training_safe_forms.py`

## Acceptance Criteria

- Add a central form catalog with explicit support tiers and a conservative `training_safe` subset.
- Make the default training dataset selection use the training-safe subset rather than every available form.
- Keep opt-in paths for `all`, `traditional`, `learnable`, and single-form runs.
- Add a generated adversarial regression harness that mutates the positive fixtures for each training-safe form and verifies the mutation fails canonically.
