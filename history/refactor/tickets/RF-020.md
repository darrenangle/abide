# RF-020: Harden Blended-Score Pass/Fail In Naani And Skeltonic

- Priority: `P1`
- Status: `done`
- Area: `world and modern forms`
- Origin: `FOLLOWUP-012`

## Problem

`Naani` and `Skeltonic` still derive canonical `passed` from a blended score threshold instead of requiring their defining structural constraints. That allows wrong line counts to report `passed=True` when the softer sub-scores are high enough.

## Primary Files

- `src/abide/forms/world.py`
- `src/abide/forms/modern.py`
- `tests/forms/`

## Acceptance Criteria

- `Naani` must require its exact four-line structure in addition to total-syllable compliance.
- `Skeltonic` must require its minimum line count in addition to the short-line and rhyme-run checks it already uses.
- Add direct regressions for the reproduced wrong-line-count false positives and valid baseline examples.
- Keep any public descriptions aligned with the behavior actually enforced.

## Notes

- Completed:
  - added direct coverage for `Naani` and `Skeltonic`
  - fixed `Naani` so canonical pass requires the exact four-line structure as well as the total-syllable range
  - fixed `Skeltonic` so canonical pass requires the minimum line count in addition to its blended short-line and rhyme-run score
