# RF-055: Harden Shape And Meter Constraint Constructors Against Degenerate Configs

- Priority: `P1`
- Status: `done`
- Area: `constraint contract validation`
- Origin: `MANUAL-VERIFIER-AUDIT-2026-04-01`

## Problem

The remaining shape and meter primitives still accept empty or incomplete
configuration and either pass vacuously or crash later. Reproduced cases
include `LineShape(lengths=[])`, `LineShape(shape_type=ShapeType.DIAMOND)`,
`LineShape(shape_type=ShapeType.DIAMOND, num_lines=0)`,
`LineLengthRange(min_length=0, max_length=0)`, `Meter(MeterType.IAMB, 0)`,
and `MeterPattern(MeterType.IAMB, [])`.

## Primary Files

- `src/abide/constraints/shape.py`
- `src/abide/constraints/meter.py`
- `tests/constraints/test_shape.py`
- `tests/constraints/test_meter.py`

## Acceptance Criteria

- Shape constraints must reject empty, missing, zero-sized, or inverted layout
  configuration instead of passing vacuously.
- Meter constraints must reject zero-foot, empty-pattern, and invalid
  probability/tolerance configuration instead of crashing later or passing with
  meaningless scores.
- Add direct regression tests for the reproduced cases.

## Completed

- Hardened `LineShape` and `LineLengthRange` so they reject empty, missing, zero-sized, or inverted layout configuration instead of constructing vacuous shape constraints.
- Hardened `Meter` and `MeterPattern` so they reject zero-foot, empty-pattern, invalid tolerance, and invalid `min_score` configuration instead of crashing later or passing with meaningless scores.
- Updated `LineLengthRange`, `Meter`, and `MeterPattern` to fail cleanly on empty poem input instead of returning vacuous `passed=True` results from empty rubrics.
- Added direct regressions in `test_shape.py` and `test_meter.py`, and verified with `uv run ruff check src/abide tests scripts`, `uv run mypy src/abide`, and `uv run pytest -q -W error` -> `772 passed, 1 skipped`.
