# RF-001: Redefine Composite Pass/Fail Semantics

- Priority: `P1`
- Status: `in_progress`
- Area: `core scoring semantics`
- Origin: `AUDIT-001`

## Problem

`WeightedSum` currently marks a composite form as passed when the weighted score clears a threshold, even if defining child constraints failed. This creates false-positive canonical passes during training and evaluation.

## Primary Files

- `src/abide/constraints/operators.py`
- `src/abide/forms/villanelle.py`
- `src/abide/forms/sonnet.py`

## Reproduction

- `Villanelle()` passes a poem with correct stanza/rhyme shape but no refrain repetition.
- `ShakespeareanSonnet()` passes a repeated-line poem that clears score threshold without satisfying the rhyme constraint.

## Acceptance Criteria

- `WeightedSum` supports hard-gating required child constraints.
- High-risk forms can opt into hard gates without losing partial reward scores.
- The non-refrain villanelle reproduction fails while positive fixtures still score reasonably.

## Notes

- Implemented:
  - `WeightedSum.required_indices`
  - `Villanelle` gating on all defining child constraints
  - sonnet-family gating for Shakespearean, Petrarchan, and Spenserian variants
  - rhyme-driven gating for couplet, quatrain, limerick, ottava-rima, and rhyme-royal variants
  - regression coverage for the villanelle false positive
  - regression coverage for the repeated-line Shakespearean false positive
- Remaining work:
  - migrate additional high-risk forms that should use gated pass semantics
  - keep meter-claim fixes in `RF-003` / `RF-004`, since those are a different class of bug
