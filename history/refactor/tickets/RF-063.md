# RF-063: Eliminate Ghazal Perfect-Score Leakage When Qafiya Evidence Is Missing

- Priority: `P1`
- Status: `done`
- Area: `reward integrity`
- Origin: `DEGENERATE-INPUT-RESCAN-2026-04-01`

## Problem

`Ghazal` still has a degenerate path where canonical `passed=False` but the
blended score remains perfect. Reproduction: a 10-line poem of identical
`"same line again"` lines returns `passed=False, score=1.0` because the full
line is detected as radif, which leaves no qafiya words to check. The form
correctly refuses to pass canonically, but the missing qafiya evidence does not
penalize the score.

## Primary Files

- `src/abide/forms/ghazal.py`
- `tests/fixtures/test_form_validation.py`

## Acceptance Criteria

- Missing qafiya evidence must materially reduce the `Ghazal` score instead of
  allowing a perfect or near-perfect result.
- The reproduced repeated-line degenerate poem must no longer score near
  perfectly.
- Valid ghazal fixtures should remain valid.

## Completed

- Added an explicit qafiya-extraction completeness check so `Ghazal` scores now
  reflect whether the verifier could actually find the pre-radif rhyme words
  the form requires.
- Penalized the overall score when qafiya evidence is missing, which removes the
  repeated-line `score=1.0` leak while preserving valid ghazal fixtures.
- Added a direct regression for the reproduced repeated-line degenerate poem.
