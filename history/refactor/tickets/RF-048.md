# RF-048: Reject Unsupported Rispetto Variants Instead Of Silently Falling Back To Tuscan

- Priority: `P1`
- Status: `done`
- Area: `world form variant validation`
- Origin: `MANUAL-VERIFIER-AUDIT-2026-04-01`

## Problem

`Rispetto(variant=\"bogus\")` currently succeeds and quietly uses the Tuscan rhyme scheme while still describing itself as `Rispetto (bogus)`. That accepts an invalid configuration and produces a verifier different from what the caller requested.

## Primary Files

- `src/abide/forms/world.py`
- `tests/forms/test_world_and_modern_forms.py`

## Acceptance Criteria

- `Rispetto` must reject unsupported variants instead of silently falling back to Tuscan.
- Supported `tuscan` and `sicilian` variants must keep working.
- Add a regression for the reproduced invalid-variant case.

## Completed

- Added explicit variant validation to `Rispetto` so unsupported variants fail fast instead of silently falling back to Tuscan behavior.
- Added a direct regression for the reproduced invalid-variant case.
- Verified with `uv run ruff check src/abide tests scripts`, `uv run mypy src/abide`, and `uv run pytest -q -W error` -> `668 passed, 1 skipped`.
