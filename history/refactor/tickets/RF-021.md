# RF-021: Align Pre-commit Ruff Hook With Project Ruff Version

- Priority: `P2`
- Status: `done`
- Area: `tooling`
- Origin: `FOLLOWUP-013`

## Problem

The repo’s direct lint gate and commit-time lint gate are using materially different Ruff versions. The project runs `ruff 0.14.8` via `uv`, but `.pre-commit-config.yaml` still pins `ruff-pre-commit` to `v0.8.4`. That mismatch keeps producing avoidable import-order churn in `scripts/param_search.py` and makes the repo’s own quality checks disagree.

## Primary Files

- `.pre-commit-config.yaml`

## Acceptance Criteria

- Update the Ruff pre-commit hook pin so it matches the Ruff version used by the project environment.
- Verify that `uv run ruff check src/abide tests scripts` and `uv run pre-commit run ruff --files scripts/param_search.py` agree on `scripts/param_search.py`.
- Keep the rest of the pre-commit behavior unchanged.

## Notes

- Completed:
  - updated `ruff-pre-commit` from `v0.8.4` to `v0.14.8` to match the project Ruff version
  - confirmed the direct Ruff gate and pre-commit Ruff hook now agree on `scripts/param_search.py`
  - normalized `scripts/param_search.py` once under the aligned formatter output
