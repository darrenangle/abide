# RF-077: Add A Resumable Longer-Canary Gemma 4 Workflow With Persisted Telemetry Artifacts

- Priority: `P2`
- Status: `done`
- Area: `training reliability`
- Origin: `GEMMA4-LONGER-CANARY-2026-04-03`

## Problem

The repo now has a working Gemma 4 E4B `vLLM` path, but operationally it is
still closer to a smoke path than a real canary workflow. The TRL trainer does
not persist reward telemetry snapshots to disk, does not expose a clean resume
path, and the Gemma runner still expects users to hand-edit prompt counts and
other knobs for each run. That makes longer canaries harder to inspect and
needlessly brittle after interruptions.

## Primary Files

- `scripts/train_grpo_trl.py`
- `scripts/reward_telemetry.py`
- `scripts/run_grpo_gemma4_e4b.sh`
- `tests/integration/`
- `README.md`

## Acceptance Criteria

- The TRL trainer must support resuming from checkpoints, including an
  auto-resume path that picks the latest checkpoint in the output directory.
- Reward telemetry for TRL runs must optionally persist structured snapshots to
  disk so longer runs can be inspected after the process exits.
- The Gemma 4 E4B runner must expose named run profiles suitable for smoke,
  canary, and longer soak-style runs without manual script edits.
- The new workflow must be covered by tests and directly verified with a longer
  live Gemma 4 E4B canary than the prior one-step smoke.

## Resolution

- Added optional JSONL persistence to `scripts/reward_telemetry.py` and wired
  `--telemetry-jsonl`, `--resume-from-checkpoint`, and `--auto-resume` into
  `scripts/train_grpo_trl.py`, including helpers that resolve the latest
  `checkpoint-*` directory in the output tree.
- Reworked `scripts/run_grpo_gemma4_e4b.sh` into a profile-driven runner with
  `smoke`, `canary`, and `soak` presets, default auto-resume, end-of-run
  telemetry summaries, and cached local-snapshot resolution so Gemma 4 model
  artifacts can load without depending on live Hub reachability.
- Added integration coverage for telemetry JSONL persistence, resume helper
  selection, the new TRL CLI flags, and the runner-script profile/local-cache
  behavior, then updated the README run-script notes accordingly.
- Verified a longer live Gemma 4 E4B canary with:
  `ABIDE_RUN_PROFILE=canary ABIDE_PORT=8129 ABIDE_OUTPUT_DIR=models/grpo_trl_gemma4_e4b_canary_rf077 bash scripts/run_grpo_gemma4_e4b.sh`
  which completed 8 steps successfully, persisted 4 telemetry snapshots to
  `models/grpo_trl_gemma4_e4b_canary_rf077/reward_telemetry.jsonl`, and saved
  `models/grpo_trl_gemma4_e4b_canary_rf077/final`.
