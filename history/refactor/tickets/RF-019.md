# RF-019: Harden Novel Semantic-Token Forms And Short-Input Handling

- Priority: `P1`
- Status: `done`
- Area: `novel forms`
- Origin: `FOLLOWUP-011`

## Problem

Several forms in `novel.py` still rely on fragile manual matching. `ColorSpectrum` and `ElementalVerse` accept raw substrings instead of whole tokens, `VowelPilgrimage` treats leading punctuation as the opening letter, and `SandwichSonnet` can raise `UnboundLocalError` on short inputs instead of returning a clean failure result.

## Primary Files

- `src/abide/forms/novel.py`
- `tests/forms/`

## Acceptance Criteria

- `ColorSpectrum` and `ElementalVerse` must match whole words rather than arbitrary substrings.
- `VowelPilgrimage` must ignore leading punctuation when checking opening vowels.
- `SandwichSonnet` must return a deterministic failing `VerificationResult` on short inputs instead of raising.
- Add direct regression coverage for the reproduced bugs plus valid baseline examples.

## Notes

- Completed:
  - added direct coverage for `ColorSpectrum`, `ElementalVerse`, `VowelPilgrimage`, and `SandwichSonnet`
  - switched color and element matching to whole-word detection instead of raw substring search
  - made `VowelPilgrimage` ignore leading punctuation when finding the opening vowel
  - fixed `SandwichSonnet` so short inputs return a clean failing result instead of raising `UnboundLocalError`
