# RF-016: Harden Fibonacci And Free-Form Structural Verifiers

- Priority: `P1`
- Status: `done`
- Area: `algorithmic form parity`
- Origin: `FOLLOWUP-008`

## Problem

The Fibonacci family still uses ungated lenient composites, so single-line syllable near misses are likely to keep enough blended score to report `passed=True`. `FreeVerse` and `ProsePoem` also expose public descriptions that talk about poetic qualities the code does not verify, and neither family currently has direct behavioral coverage.

## Primary Files

- `src/abide/forms/fibonacci.py`
- `src/abide/forms/free_verse.py`
- `tests/forms/`

## Acceptance Criteria

- Add direct coverage for `FibonacciPoem`, `ReverseFibonacci`, `DoubleFibonacci`, `FreeVerse`, and `ProsePoem`.
- Ensure lenient Fibonacci forms keep partial score without reporting canonical `passed=True` on defining-pattern near misses.
- Narrow public descriptions in the free-form family to structural properties the verifier actually checks.

## Notes

- Completed:
  - added direct coverage for `FibonacciPoem`, `ReverseFibonacci`, `DoubleFibonacci`, `FreeVerse`, and `ProsePoem`
  - hardened the Fibonacci family with `WeightedSum.required_indices` so lenient near misses no longer pass canonically
  - fixed a structural bug in `ProsePoem` so line-broken verse no longer passes as paragraph-form prose
  - narrowed the free-form module/class descriptions to structural properties the code actually checks
