# RF-022: Harden Exact-count Constrained Forms And FreeVerse Word-bound Semantics

- Priority: `P1`
- Status: `done`
- Area: `constrained and free-form verifiers`
- Origin: `FOLLOWUP-014`

## Problem

Some remaining verifiers still use lenient reward shaping in ways that undermine canonical correctness. `Abecedarian` and `Mesostic` can report `passed=True` when the poem is missing the final required lines because they only grade the provided prefix. `FreeVerse` can also report `passed=True` while violating configured per-line word-count bounds because those violations only apply a mild score multiplier and do not affect canonical pass/fail.

## Primary Files

- `src/abide/forms/constrained.py`
- `src/abide/forms/free_verse.py`
- `tests/forms/`

## Acceptance Criteria

- `Abecedarian` and `Mesostic` must fail canonically when required lines are missing.
- Their scoring should reflect missing required lines as real violations rather than grading only the available prefix.
- `FreeVerse` must fail canonically when configured per-line minimum or maximum word bounds are violated.
- Add direct regressions for the reproduced false positives plus valid baseline examples.

## Notes

- Completed:
  - added direct regressions for missing-line `Abecedarian` and `Mesostic` false positives
  - changed both exact-count constrained forms so missing required lines count as real violations and canonical pass requires the full expected match count
  - tightened `FreeVerse` so configured per-line word-bound violations are recorded in `issues` and block canonical pass
