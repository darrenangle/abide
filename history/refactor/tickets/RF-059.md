# RF-059: Penalize Short-Prefix Leaks In Non-Uniform Word-Count Patterns

- Priority: `P1`
- Status: `done`
- Area: `reward integrity`
- Origin: `DEGENERATE-INPUT-AUDIT-2026-04-01`

## Problem

The `WordCount` primitive still scores short poems against only the lines that
exist, which means any form built on a non-uniform exact pattern can receive
near-perfect reward from a prefix that matches only the opening of the target
sequence. Reproduced examples include `PrimeVerse("alpha beta")` (`0.904`),
`HourglassVerse("alpha")` (`0.900`), `DescendingStaircase("one two three four five six seven")`
(`0.901`), `NumericalEcho("one two three")` (`0.900`), `BinaryBeat("alpha beta")`
(`0.900`), and `GoldenRatio("alpha")` (`0.904`).

## Primary Files

- `src/abide/constraints/lexical.py`
- `tests/constraints/test_lexical.py`
- `tests/forms/test_novel_forms.py`

## Acceptance Criteria

- Non-uniform `WordCount` patterns must penalize missing expected lines instead
  of grading only the present prefix.
- The reproduced prefix inputs must no longer score near-perfectly.
- Keep current behavior for uniform per-line counts where the constraint is
  intentionally open-ended.

## Completed

- Updated non-uniform `WordCount` scoring to count missing expected lines
  against the score instead of evaluating only the present prefix.
- Added direct primitive-level regressions for missing-line scoring and for the
  preserved uniform-count behavior.
- Added form-level regressions for the reproduced exact-pattern prefix leaks in
  `PrimeVerse`, `HourglassVerse`, `DescendingStaircase`, `NumericalEcho`,
  `BinaryBeat`, and `GoldenRatio`.
