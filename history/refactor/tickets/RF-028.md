# RF-028: Stop Skeltonic From Treating Repeated Identical End Words As Valid Rhyme Runs

- Priority: `P1`
- Status: `done`
- Area: `modern form heuristics`
- Origin: `FOLLOWUP-020`

## Problem

`Skeltonic.verify()` currently counts a run of identical end words as a valid rhyme run because it only compares end sounds. A poem with ten short lines all ending in the exact same word can therefore return `passed=True`, which is the same loophole class that earlier rhyme constraints already closed.

## Primary Files

- `src/abide/forms/modern.py`
- `tests/forms/`

## Acceptance Criteria

- `Skeltonic` must not treat repeated identical end words as evidence of a rhyme run.
- A short-line poem that games the form by repeating the exact same end word must no longer pass canonically.
- Keep legitimate short-line/rhyme-run behavior intact for non-identical rhyming end words.
- Add direct regressions for the reproduced false positive and a valid contrasting example.

## Completed

- Updated `Skeltonic.verify()` to reject rhyme-run extensions when the end word itself is unchanged, even if the end sound matches.
- Replaced the prior positive fixture with non-identical rhyming end words and added a regression for repeated identical end-word abuse.
- Verified with `uv run ruff check src/abide tests scripts`, `uv run mypy src/abide`, and `uv run pytest` -> `605 passed, 1 skipped`.
