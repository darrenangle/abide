# RF-045: Remove The Hidden 20-Word Gate From UniqueUtterance

- Priority: `P1`
- Status: `done`
- Area: `novel form contract alignment`
- Origin: `MANUAL-VERIFIER-AUDIT-2026-04-01`

## Problem

`UniqueUtterance` claims only a minimum line count plus no repeated words, but it delegates to `AllWordsUnique(min_words=20)`. That creates a hidden extra requirement: a 6-line poem with 18 unique words can report `score=1.0` and still fail canonically.

## Primary Files

- `src/abide/forms/novel.py`
- `tests/forms/test_novel_forms.py`

## Acceptance Criteria

- `UniqueUtterance` must not require an undocumented 20-word minimum.
- A 6-line poem with all unique words should pass when it satisfies the stated line-count requirement.
- Add a direct regression for the reproduced hidden-threshold failure.

## Completed

- Removed the hidden `min_words=20` gate from `UniqueUtterance` so the form now matches its public contract: minimum line count plus no repeated words.
- Added a direct regression for the reproduced six-line all-unique poem that previously scored perfectly but still failed canonically.
- Verified with `uv run ruff check src/abide tests scripts`, `uv run mypy src/abide`, and `uv run pytest -q -W error` -> `664 passed, 1 skipped`.
