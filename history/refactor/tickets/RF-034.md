# RF-034: Fix Stale Special-Form Kwargs So The Catalog Instantiates Every Exported Form

- Priority: `P1`
- Status: `done`
- Area: `catalog coverage`
- Origin: `SWEEP-2026-04-01`

## Problem

`load_form_instances()` still skips 10 exported forms because several `SPECIAL_FORM_KWARGS` entries no longer match the actual constructor signatures. That undermines automated catalog sweeps, tooling, and any code that assumes the public form surface can be instantiated from the helper layer.

## Primary Files

- `src/abide/forms/catalog.py`
- `tests/forms/`

## Acceptance Criteria

- `instantiate_form()` must succeed for every exported form name in `abide.forms.__all__`.
- Correct the stale special kwargs to match the real constructor signatures.
- Add a regression that fails if `load_form_instances()` silently omits exported forms again.

## Completed

- Updated the stale special kwargs to match the real constructor signatures for the remaining hard and mathematical forms.
- Added a regression that asserts `load_form_instances()` covers the full exported form set.
- Verified with `uv run ruff check src/abide tests scripts`, `uv run mypy src/abide`, and `uv run pytest -q -W error` -> `612 passed, 1 skipped`.
