# RF-015: Harden Japanese Short Forms And Shape-Poetry Families

- Priority: `P1`
- Status: `done`
- Area: `form parity / uncovered exports`
- Origin: `FOLLOWUP-007`

## Problem

Several exported short-form families still have little or no direct behavioral coverage, and at least some of them retain the old lenient-pass problem where a near miss can keep a high enough score to report `passed=True`. Some public descriptions in this group also claim semantics the verifier does not actually check.

## Primary Files

- `src/abide/forms/japanese.py`
- `src/abide/forms/diamante.py`
- `tests/forms/`
- `tests/fixtures/`

## Acceptance Criteria

- Add direct behavioral coverage for `Senryu`, `Katauta`, `Sedoka`, `Diamante`, `Cinquain`, `WordCinquain`, `Etheree`, and `ReverseEtheree`.
- Ensure lenient mode keeps partial reward without reporting canonical `passed=True` on defining-structure near misses.
- Narrow any public descriptions in this family to what the verifier actually checks.

## Notes

- Completed:
  - added direct coverage for `Senryu`, `Katauta`, `Sedoka`, `Diamante`, `Cinquain`, `WordCinquain`, `Etheree`, and `ReverseEtheree`
  - hardened Japanese short forms so lenient mode preserves partial score but never reports `passed=True` when a defining syllable line is wrong
  - hardened shape-poetry families with `WeightedSum.required_indices` so one broken shape line cannot pass canonically in lenient mode
  - narrowed `Senryu` and shape-form docstrings/descriptions to the structural properties the verifier actually checks
