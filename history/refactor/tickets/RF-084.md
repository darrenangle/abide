# RF-084: Add Mixed-Stable Multi-Seed Validation and Holdout Export

- Priority: `P1`
- Status: `done`
- Area: `training reliability`
- Origin: `PRIME-RL-VALIDATION-2026-04-10`

## Problem

`RF-083` produced a strong mixed-form Gemma 4 E2B baseline, but the repo still
does not have a first-class way to validate that baseline across multiple seeds
or export holdout prompts/samples from the trained checkpoint. The next
reliability step is to make the mixed-stable run repeatable, compare it across
seeds, and save scored holdout outputs as durable artifacts instead of relying
on aggregate summaries alone.

## Primary Files

- `scripts/`
- `src/abide/training/`
- `tests/integration/`
- `history/refactor/`

## Acceptance Criteria

- The shell runner must expose seed override support for modern prime-rl runs.
- The repo must expose a repeatable validation path that can run multiple
  mixed-stable seeds and export scored holdout samples from the final checkpoint.
- At least one multi-seed validation pass must be executed and recorded in the
  tracker with artifact paths and metrics.

## Progress

- Added `ABIDE_SEED` passthrough to `scripts/run_prime_rl_gemma4_e2b.sh`.
- Added `build_prime_rl_prompt_records(...)` plus export coverage so prompt sets can
  be reused deterministically outside the training env package.
- Added `scripts/prime_rl_validate.py`, which can:
  run multi-seed `mixed-stable` validation,
  rebuild a loadable model artifact from the final filesystem broadcast,
  export scored `holdout_samples.jsonl`,
  and aggregate holdout metrics into per-seed plus cross-seed summaries.
- Fixed the validation overlay bug after the first live pass: evaluation shards
  were symlinked with relative paths, which made `vLLM` fail to open
  `pytorch_model-*.bin` from the temporary eval artifact. The validator now uses
  resolved absolute symlinks and supports `--evaluate-only` so finished runs can
  be rescored without retraining.
- Recorded the first multi-seed validation pass at
  `models/prime_rl_gemma4_e2b_validation_rf084/`.
  Artifacts:
  `models/prime_rl_gemma4_e2b_validation_rf084/results.jsonl`,
  `models/prime_rl_gemma4_e2b_validation_rf084/summary.json`,
  `models/prime_rl_gemma4_e2b_validation_rf084/seed_42/validation/holdout_samples.jsonl`,
  `models/prime_rl_gemma4_e2b_validation_rf084/seed_43/validation/holdout_samples.jsonl`.
- Validation outcome:
  mean training reward `0.9417`,
  mean holdout score `0.8071`,
  mean holdout pass rate `0.375`.
  Stable strong forms: `Haiku`, `Limerick`, `ShakespeareanSonnet`.
  Persistent weak forms: `Sestina`, `Tanka`, `Ghazal`.
  Unstable form: `Villanelle`.

## Validation Summary

- Seed `42`: training reward `0.8833`; holdout score `0.8115`; holdout pass rate `0.375`.
- Seed `43`: training reward `1.0000`; holdout score `0.8026`; holdout pass rate `0.375`.
- Cross-seed weak-form failure modes from exported holdout samples:
  `Ghazal` missed radif/matla structure in both seeds.
  `Tanka` repeatedly missed exact syllable counts on lines `1/2/4/5`.
  `Sestina` consistently broke stanza-two end-word cycling and once missed the envoy line count.
  `Villanelle` was perfect in seed `42` but missed refrain placement and stanza size in one seed-`43` sample.

## Acceptance

- Met. The runner exposes seed override support, the repo now has a repeatable
  multi-seed validation/export path, and the first two-seed `mixed-stable`
  validation pass is recorded with durable artifacts and aggregate metrics.
