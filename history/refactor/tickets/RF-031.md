# RF-031: Gate Generic `Sonnet` Canonical Passes On The Syllable Proxy Instead Of Line Count Alone

- Priority: `P1`
- Status: `done`
- Area: `sonnet-family pass semantics`
- Origin: `SWEEP-2026-04-01`

## Problem

The generic `Sonnet` verifier currently uses a lenient weighted threshold with no required children. As a result, any 14-line poem can pass canonically even when every line misses the stated "about 10 syllables each" proxy.

## Primary Files

- `src/abide/forms/sonnet.py`
- `tests/forms/`

## Acceptance Criteria

- Generic `Sonnet` must not report `passed=True` unless both 14-line structure and the syllable proxy pass.
- Keep partial credit behavior for near misses so reward shaping still works.
- Add a regression for the reproduced 7-syllable false positive and a valid contrasting example.

## Completed

- Added required-child gating to the lenient `WeightedSum` so both line count and the 10-syllable proxy must pass for `passed=True`.
- Added regressions for a valid 10-syllable sonnet shell and the reproduced 7-syllable false positive.
- Verified with `uv run ruff check src/abide tests scripts`, `uv run mypy src/abide`, and `uv run pytest` -> `611 passed, 1 skipped`.
