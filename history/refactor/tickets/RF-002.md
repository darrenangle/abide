# RF-002: Add Adversarial Regression Tests For False Positives

- Priority: `P1`
- Status: `in_progress`
- Area: `test coverage`
- Origin: `AUDIT-006`

## Problem

The current tests are heavily positive-example driven and miss the false positives uncovered during training-oriented audit work.

This work should be organized as an edge-case matrix by form family so related variants inherit the same adversarial expectations.

## Primary Files

- `tests/forms/test_forms.py`
- `tests/fixtures/test_form_validation.py`
- `tests/verifiers/`

## Acceptance Criteria

- Add negative tests for:
  - non-refrain villanelle
  - repeated-line sonnet false positive
  - rhymed blank verse false positive
  - ghazal with a dangling odd line
  - TRL form identity mismatch
- New tests fail before the fixes and pass after.
