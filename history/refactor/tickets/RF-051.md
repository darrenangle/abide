# RF-051: Reject Degenerate Zero-Valued Hard-Form Configurations

- Priority: `P1`
- Status: `done`
- Area: `hard form constructor validation`
- Origin: `MANUAL-VERIFIER-AUDIT-2026-04-01`

## Problem

Several `hard.py` forms accept zero-valued or empty constructor arguments and become vacuous or nonsensical verifiers. Reproductions included `StaircasePoem(num_words=0)`, `DescendingStaircasePoem(num_words=0)`, `DoubleAcrosticPoem(first_word=\"\", last_word=\"\")`, `TotalCharacterPoem(total_chars=0, min_lines=0)`, and `ExactWordPoem(word_count=0, min_lines=0)`, some of which can pass empty input.

## Primary Files

- `src/abide/forms/hard.py`
- `tests/forms/test_hard_forms.py`

## Acceptance Criteria

- Hard forms must reject zero-valued or empty constructor arguments that create degenerate verifiers.
- Existing valid default constructors must keep working.
- Add direct regressions for the reproduced degenerate-constructor cases.

## Completed

- Added explicit positive-value validation across the degenerate hard-form constructors so zero-valued counts and empty acrostic words now fail fast instead of creating vacuous or nonsensical verifiers.
- Added direct regressions for the reproduced zero-valued constructor cases across the hard-form family.
- Verified with `uv run ruff check src/abide tests scripts`, `uv run mypy src/abide`, and `uv run pytest -q -W error` -> `686 passed, 1 skipped`.
