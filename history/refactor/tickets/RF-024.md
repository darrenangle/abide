# RF-024: Remove Permanent Trust-tier Framing From The Form Catalog And RL Defaults

- Priority: `P2`
- Status: `done`
- Area: `catalog, scripts, public docs`
- Origin: `FOLLOWUP-016`

## Problem

The repo still bakes a permanent trust taxonomy into the form catalog and training scripts through names like `training_safe`, `best_effort`, and `experimental`. That conflicts with the stated direction of bringing the full exported catalog up to the same bar. The current curated RL default is still useful operationally, but it should be represented as a temporary rollout default rather than a lasting support-tier ontology.

## Primary Files

- `src/abide/forms/catalog.py`
- `scripts/prompt_generator.py`
- `scripts/train_grpo.py`
- `scripts/train_grpo_trl.py`
- `README.md`
- `tests/`

## Acceptance Criteria

- Remove the public support-tier API and its `training_safe` / `best_effort` / `experimental` framing from active codepaths.
- Replace the curated training subset naming with a neutral RL-default concept in catalog helpers, scripts, tests, and README usage examples.
- Preserve the current curated subset and tuned defaults behavior for RL scripts.
- Keep all repo verification gates green after the rename/refactor.

## Notes

- Completed:
