# RF-056: Harden Remaining Lexical Constraint Constructors And 1-Based Position Contracts

- Priority: `P1`
- Status: `done`
- Area: `constraint contract validation`
- Origin: `MANUAL-VERIFIER-AUDIT-2026-04-01`

## Problem

The remaining public lexical constraints still accept empty patterns, zero
bounds, invalid 1-based positions, or malformed target values and either pass
vacuously or crash later. Reproduced cases include `LineStartsWith([])`,
`LineStartsWith(\"\")`, `LineEndsWith([])`, `VowelConsonantPattern(\"\")`,
`ExactTotalCharacters(0)`, `WordLengthStaircase(max_words=0)`,
`NoSharedLetters([(0, 1)])`, `DoubleAcrostic(\"\", \"\")`,
`ExactCharacterBudget(\"e\", 0)`, and `PositionalCharacter([(0, \"a\")])`.

## Primary Files

- `src/abide/constraints/lexical.py`
- `tests/constraints/test_lexical.py`

## Acceptance Criteria

- Remaining lexical constraint constructors must reject empty patterns, zero
  budgets, malformed string targets, and invalid pair/position configuration
  instead of crashing later or passing vacuously.
- Constraints that document 1-based positions or line numbers must reject `0`
  and negative values instead of relying on Python negative indexing.
- Add direct regressions for the reproduced vacuous-pass and runtime-error
  cases.

## Completed

- Hardened the remaining public lexical primitives so they now reject empty patterns, zero budgets, malformed string targets, invalid percentage ranges, and invalid 1-based positions or line numbers instead of crashing later or passing vacuously.
- Fixed 1-based contract violations in `NoSharedLetters` and `PositionalCharacter`, plus empty-target and zero-budget issues in `LineStartsWith`, `LineEndsWith`, `LetterFrequency`, `NoConsecutiveRepeats`, `VowelConsonantPattern`, `ExactTotalCharacters`, `ExactTotalVowels`, `WordLengthStaircase`, `CrossLineVowelWordCount`, `ExactWordCount`, `DoubleAcrostic`, and `ExactCharacterBudget`.
- Added direct constructor regressions for the reproduced lexical misconfiguration cases and verified with `uv run ruff check src/abide tests scripts`, `uv run mypy src/abide`, and `uv run pytest -q -W error` -> `796 passed, 1 skipped`.
