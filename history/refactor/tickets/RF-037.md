- Priority: `P2`

## Problem

After fixing `Pantoum`, the repo still relies on ad hoc checks for the broader stanza-flattening regression class. Multi-stanza fixed forms should not keep passing when their blank-line stanza boundaries are collapsed into a single block, unless the form explicitly allows that presentation.

## Primary Files

- `tests/forms/`
- `tests/fixtures/poems.py`

## Acceptance Criteria

- Add a shared flattening mutation harness for representative multi-stanza fixed forms.
- Assert the baseline fixture passes and the flattened version fails canonically for stanza-sensitive forms.
- Exclude forms intentionally documented to allow single-block presentation.

## Completed

- Added shared stanza-flattening regressions for representative multi-stanza fixed forms.
- Asserted that each canonical baseline passes and the flattened single-block mutation fails canonically.
