# RF-054: Harden Public Constraint Constructors Against Empty Patterns And Invalid Indices

- Priority: `P1`
- Status: `done`
- Area: `constraint contract validation`
- Origin: `MANUAL-VERIFIER-AUDIT-2026-04-01`

## Problem

Several public constraints still accept empty patterns, empty target
collections, or invalid indices and either pass vacuously or fail with runtime
errors. Reproduced cases include `RhymeScheme(\"\")`, `Refrain(..., repeat_at=[])`,
`LinePairSimilarity([])`, `EndWordPattern(num_words=0, num_stanzas=0)`,
`WordCount([])`, `ForcedWords([])`, `WordLengthPattern([])`, `Acrostic(\"\")`,
`StanzaSizes([])`, and `SyllablesPerLine([])`.

## Primary Files

- `src/abide/constraints/relational.py`
- `src/abide/constraints/structural.py`
- `src/abide/constraints/lexical.py`
- `tests/constraints/test_relational.py`
- `tests/constraints/test_structural.py`
- `tests/constraints/test_lexical.py`

## Acceptance Criteria

- Public constraint constructors must reject empty structural patterns,
  empty target collections, and invalid negative line indices.
- Rotation- and pair-based relational constraints must validate shape and
  permutation invariants instead of crashing later or succeeding vacuously.
- Add direct regression tests for the reproduced vacuous-pass and runtime-error
  cases.

## Completed

- Added shared constraint-constructor validation helpers and used them to reject empty structural patterns, empty target collections, invalid negative indices, invalid rotations, and out-of-range threshold values across the structural, lexical, and relational primitives.
- Hardened `LineCount`, `StanzaCount`, `StanzaSizes`, `SyllablesPerLine`, `WordCount`, `ForcedWords`, `WordLengthPattern`, `RhymeScheme`, `Refrain`, `LinePairSimilarity`, `OpeningPhraseRefrain`, `EndRhymePairs`, `EndWordPattern`, and `Acrostic` against vacuous or crash-prone constructor inputs.
- Updated dynamic-form call sites in `BlankVerse`, `Pantoum`, `BluesPoem`, and `TerzaRima` so malformed poem input now fails cleanly instead of constructing invalid internal constraints at runtime.
- Added direct constructor regressions in the constraints test suite and verified with `uv run ruff check src/abide tests scripts`, `uv run mypy src/abide`, and `uv run pytest -q -W error` -> `755 passed, 1 skipped`.
