# RF-079: Retarget The Legacy Verifiers Gemma Trainer To Verified Gemma 4 E4B Defaults

- Priority: `P2`
- Status: `done`
- Area: `training reliability`
- Origin: `VERIFIERS-GEMMA4-RETARGET-2026-04-03`

## Problem

`RF-078` refreshed the legacy verifiers training path and rewrote it around a
well-known-form subset, but the default model and runner path still target
Gemma 3. The user requirement was to make that refreshed verifiers path train
Gemma 4, not just to modernize the script shape. We need to verify what the
isolated `.venv-verifiers` runtime can actually support for Gemma 4 E4B, add
any runtime overlay needed, and retarget the defaults and runner docs once the
Gemma 4 path is directly proven.

## Primary Files

- `scripts/train_grpo.py`
- `scripts/run_grpo.sh`
- `scripts/run_grpo_gemma3.sh`
- `scripts/prepare_verifiers_runtime.sh`
- `scripts/model_profiles.py`
- `tests/integration/`

## Acceptance Criteria

- The isolated legacy verifiers runtime must directly verify Gemma 4 E4B model
  loading and `vf-vllm` serving, or encode the exact overlay needed for that
  path.
- The legacy verifiers trainer defaults and primary Gemma runner must target
  Gemma 4 E4B rather than Gemma 3.
- The updated Gemma 4 legacy-verifiers path must be covered by tests and direct
  local verification.

## Completed

- Added a Gemma 4-compatible legacy vLLM wrapper in `src/abide/verifiers_vllm_server.py`
  and updated the isolated runtime prep to pin the verified Gemma 4 overlay,
  remove the incompatible `flash-attn`, and install `bitsandbytes` for the
  memory-constrained legacy trainer path.
- Updated `scripts/train_grpo.py` to default to Gemma 4 E4B, bridge the newer
  verifiers client API, patch the legacy PEFT prep for k-bit models, and apply
  verified Gemma 4 runtime defaults when the user does not override them.
- Verified the end-to-end legacy Gemma 4 path locally with:
  - `bash scripts/prepare_verifiers_runtime.sh`
  - `CUDA_VISIBLE_DEVICES=1 .venv-verifiers/bin/python -m abide.verifiers_vllm_server --model google/gemma-4-E4B-it --port 8133 --gpu-memory-utilization 0.80 --tensor-parallel-size 1 --max-model-len 1024 --trust-remote-code --disable-log-stats --enforce-eager --dtype bfloat16`
  - `CUDA_VISIBLE_DEVICES=0 .venv-verifiers/bin/python scripts/train_grpo.py --model google/gemma-4-E4B-it --form-set well_known --prompts 1 --save-steps 10 --output models/abide_verifiers_gemma4_e4b_legacy_smoke_defaulted --port 8133 --no-wandb`
  which completed successfully and saved `models/abide_verifiers_gemma4_e4b_legacy_smoke_defaulted/final`.
