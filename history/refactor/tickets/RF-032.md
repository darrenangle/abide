# RF-032: Make The Test Suite Warning-Clean Under `pytest -W error`

- Priority: `P2`
- Status: `done`
- Area: `test harness`, `dependency noise`
- Origin: `SUITE-WARNINGS-2026-04-01`

## Problem

The full suite is functionally green but still emits two warnings on every run:

- a `HypothesisDeprecationWarning` from our `tests/conftest.py` profile using `verbosity=2`
- a `UserWarning` from the `pronouncing` dependency importing `pkg_resources`

This leaves the repo in an ambiguous "green with warnings" state instead of a clean pass signal.

## Primary Files

- `tests/conftest.py`
- `pyproject.toml`

## Acceptance Criteria

- Replace the deprecated Hypothesis verbosity setting with the current supported API.
- Filter the known upstream `pronouncing` `pkg_resources` warning in pytest configuration.
- `uv run pytest -q -W error` must pass cleanly.

## Completed

- Updated `tests/conftest.py` to use `Verbosity.verbose` instead of the deprecated integer verbosity value.
- Suppressed the upstream `pkg_resources` warning at the `pronouncing` import boundary inside `src/abide/primitives/phonetics.py`, which keeps the warning local and allows strict warning runs to stay clean.
- Verified with `uv run ruff check src/abide tests scripts`, `uv run mypy src/abide`, and `uv run pytest -q -W error` -> `611 passed, 1 skipped`.
