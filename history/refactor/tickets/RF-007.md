# RF-007: Declare And Isolate TRL Training Dependencies

- Priority: `P2`
- Status: `done`
- Area: `packaging / entrypoints`
- Origin: `AUDIT-005`

## Problem

`scripts/train_grpo_trl.py` imports heavyweight training dependencies at module import time, and the declared extras do not cleanly match what the script needs.

## Primary Files

- `scripts/train_grpo_trl.py`
- `pyproject.toml`

## Acceptance Criteria

- Training-only imports move into runtime paths where practical.
- Project metadata exposes an install target that actually supports the TRL script.
- Helper functions stay importable in lighter environments.

## Resolution

- Heavy training imports moved into `main()`.
- Added explicit TRL-path dependencies to the `training` extra in `pyproject.toml`.
- Confirmed the script can now be imported in the current environment without the full runtime stack.
