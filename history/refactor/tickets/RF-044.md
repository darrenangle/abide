# RF-044: Require Whole-Token Prefix Matches For Explicit Anaphora Phrases

- Priority: `P1`
- Status: `done`
- Area: `anaphora phrase matching`
- Origin: `MANUAL-VERIFIER-AUDIT-2026-04-01`

## Problem

`Anaphora(phrase=...)` currently counts lines with longer prefixed words as exact phrase matches because it uses raw `startswith()`. For example, `I have a dreamer waiting` satisfies the explicit phrase `I have a dream`, which is not a real repeated phrase match.

## Primary Files

- `src/abide/forms/constrained.py`
- `tests/forms/test_constrained_forms.py`

## Acceptance Criteria

- Explicit anaphora phrases must match whole leading tokens, not raw string prefixes.
- Existing valid explicit and auto-detected anaphora behavior must keep working.
- Add a direct regression for the reproduced `dream`/`dreamer` false positive.

## Completed

- Switched explicit anaphora phrase matching to whole-token prefix comparison and normalized auto-detection onto the same tokenization path.
- Added a direct regression for the reproduced `dream` / `dreamer` false positive while preserving the existing auto-detected opening behavior.
- Verified with `uv run ruff check src/abide tests scripts`, `uv run mypy src/abide`, and `uv run pytest -q -W error` -> `663 passed, 1 skipped`.
