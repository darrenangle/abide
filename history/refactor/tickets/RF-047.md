# RF-047: Replace Silent Line-Count Clamps With Early Validation In Bounded Pattern Forms

- Priority: `P1`
- Status: `done`
- Area: `constructor contract validation`
- Origin: `MANUAL-VERIFIER-AUDIT-2026-04-01`

## Problem

Several bounded forms silently clamp unsupported `num_lines` values instead of rejecting them. Examples: `NumberWord(12)` becomes a 10-line verifier, `PiKu(25)` becomes 20 lines, and `SelfReferential(12)` becomes 9 lines. That means callers can request one verifier and quietly get another.

## Primary Files

- `src/abide/forms/novel.py`
- `src/abide/forms/mathematical.py`
- `tests/forms/test_novel_forms.py`
- `tests/forms/test_mathematical_forms.py`

## Acceptance Criteria

- `NumberWord`, `PiKu`, and `SelfReferential` must reject unsupported line counts instead of silently clamping them.
- Supported configurations must keep working.
- Add direct regressions for the reproduced silent-clamp cases.

## Completed

- Replaced silent line-count clamps with early validation in `NumberWord`, `PiKu`, and `SelfReferential` so callers can no longer request one verifier and quietly get another.
- Added direct regressions for the reproduced `NumberWord(12)`, `PiKu(25)`, and `SelfReferential(12)` silent-clamp cases.
- Verified with `uv run ruff check src/abide tests scripts`, `uv run mypy src/abide`, and `uv run pytest -q -W error` -> `667 passed, 1 skipped`.
