# RF-088: Build a Hard-Form SFT Warmup and RL Continuation Pipeline for Gemma

- Priority: `P1`
- Status: `in_progress`
- Area: `training curriculum`
- Origin: `user request`
- Depends On: `RF-085`, `RF-086`, `RF-087`

## Problem

The modern Gemma 4 E2B path is stable, but the current mixed-form RL recipe is
still weak on several structurally difficult forms. We now have a verifier-gated
SFT export path, so the next step is to use it as a real warm start instead of
continuing to rely on RL alone.

The requested path is:

1. identify a measured subset of difficult forms,
2. generate at least 1000 verifier-clean SFT warmup examples for that subset,
3. run a LoRA SFT warmup on Gemma,
4. continue RL from the warmed model on the same difficult-form subset.

## Primary Files

- `src/abide/training/`
- `scripts/`
- `data/sft/`
- `history/refactor/`

## Acceptance Criteria

- The difficult-form subset is grounded in recorded validation or holdout data,
  not hand-picked intuition alone.
- The repo can export a verifier-gated hard-form SFT corpus with at least 1000
  rows.
- The repo exposes a repeatable Gemma LoRA SFT warmup entrypoint for that data.
- The repo exposes a repeatable RL continuation path that trains on the hard-form
  subset starting from the SFT output.
- At least one end-to-end warmup-plus-RL probe is run and recorded in the tracker.

## Progress

- Added a codex-spark-authored hard-form warmup task/validation loop in
  `src/abide/training/codex_spark_warmup.py` plus CLI entrypoints under
  `scripts/`.
- Generated `25` codex-spark-authored candidates for each hard form in the
  current difficult subset: `Tanka`, `PetrarchanSonnet`, `Villanelle`,
  `Ghazal`, and `Sestina`.
- Tightened validation beyond exact-duplicate rejection:
  - reject rows too close to the same-form seed corpus
  - reject rows too close to already accepted same-form peers
- Exported the validated chat-SFT artifact at
  `data/sft/codex_spark_hard_form_validated.jsonl` with `125` verifier-clean
  rows and summary metadata in
  `data/sft/codex_spark_hard_form_validated.summary.json`.
- Current observed maxima for the accepted corpus:
  - `max_seed_similarity = 0.5918`
  - `max_peer_similarity = 0.8030`
