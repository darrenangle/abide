## Problem

The README makes concrete claims about metrical analysis and meter-aware verification, but the repo currently has no dedicated meter/scansion tests and at least one documented example does not match runtime behavior.

## Primary Files

- `src/abide/primitives/meter.py`
- `src/abide/constraints/meter.py`
- `tests/primitives/`
- `tests/constraints/`

## Reproduction

## Acceptance Criteria

- Add direct regression coverage for the meter primitives and meter constraints.
- Either fix the documented scansion behavior or narrow the README/docs to match what the implementation can actually guarantee.
- Ensure any public examples used in docs are executable and match real output.
