- Priority: `P2`
- Status: `done`
- Area: `training environment`
- Origin: `GEMMA4-LIVE-BOOT-2026-04-02`

## Problem

The new Gemma 4 E4B runner still launches `trl` from
`/home/darren/miniconda3/bin/trl`. On this machine that external environment
has an older Transformers build that does not recognize the `gemma4`
architecture, so the first live vLLM boot fails before the model can load.

## Primary Files

- `scripts/run_grpo_gemma4_e4b.sh`

## Acceptance Criteria

- The Gemma 4 E4B runner must use the project-managed `uv` environment for
  both `trl vllm-serve` and the training entrypoint.
- The live Gemma 4 E4B vLLM boot must get past the previous
  `Transformers does not recognize gemma4` failure.

## Resolution

- Updated `scripts/run_grpo_gemma4_e4b.sh` to launch both `trl vllm-serve` and
  `train_grpo_trl.py` via `uv run` instead of the stale external miniconda
  binaries.
- Confirmed that the live vLLM boot moved past the earlier stale-toolchain
  `gemma4` architecture failure and now reaches the current upstream vLLM
  architecture-support boundary instead.
