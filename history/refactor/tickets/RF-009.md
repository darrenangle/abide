# RF-009: Refactor Remaining Manual Fixed-Form Verifiers

- Priority: `P1`
- Status: `done`
- Area: `form verification consistency`
- Origin: `FOLLOWUP-001`

## Problem

Several fixed-form verifiers still use hand-written `score` and `passed` logic instead of the shared constraint machinery. That keeps the last high-risk drift surface alive: a form can change behavior independently of the core composite semantics, and new false positives are harder to test systematically.

## Primary Files

- `src/abide/forms/ballade.py`
- `src/abide/forms/rondeau.py`
- `src/abide/forms/pantoum.py`
- `src/abide/forms/terza_rima.py`
- `src/abide/forms/blues.py`
- `src/abide/forms/clerihew.py`
- `src/abide/constraints/relational.py`
- `src/abide/constraints/structural.py`

## Reproduction

- `Pantoum()` previously passed a poem with correct quatrain count and inter-stanza repetition but no circular close.
- `TerzaRima()` previously passed a poem whose final tail line broke the closing chain rhyme.
- `Clerihew()` previously passed a rhymed 4-liner whose opening line was not a name.
- `BluesPoem()` previously passed AAB tercets with rhyme but without the defining line-1/line-2 repetition.

## Acceptance Criteria

- The remaining fixed forms use shared constraints for structure, refrain/repetition, and rhyme checks where possible.
- Canonical `passed` is gated on the defining checks, while partial score remains available for reward shaping.
- Regressions cover the reproduced high-score near misses and the new shared primitives.
- Positive fixtures for the migrated forms still score reasonably.
