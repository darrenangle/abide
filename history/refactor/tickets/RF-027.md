# RF-027: Harden Epigram So 3-line Prose Blocks No Longer Pass As Perfect Fits

- Priority: `P1`
- Status: `done`
- Area: `short-form structural proxies`
- Origin: `FOLLOWUP-019`

## Problem

`Epigram.verify()` currently auto-passes any 3-line poem that stays within the 2-4 line count range. That means obviously essay-like three-line prose blocks can return `passed=True, score=1.0`, which is too weak even for a structural proxy.

## Primary Files

- `src/abide/forms/epigram.py`
- `tests/forms/`

## Acceptance Criteria

- `Epigram` must enforce a short-line proxy across all supported line counts.
- A long three-line prose block must no longer return `passed=True` or `score=1.0`.
- Keep the existing rhyme behavior for the 2-line and 4-line variants.
- Add direct regressions for the reproduced false positive and valid short-form examples.

## Notes

- Completed:
  - added a short-line word-count proxy across all supported epigram variants
  - preserved the 2-line and 4-line rhyme behavior while eliminating the auto-pass 3-line branch
  - added direct regressions for the reproduced essay false positive and compact baseline examples
