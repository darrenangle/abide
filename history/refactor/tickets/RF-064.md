# RF-064: Penalize Repeated-Line Overscore Leakage In Triolet

- Priority: `P1`
- Status: `done`
- Area: `reward integrity`
- Origin: `REPEATED-LINE-RESCAN-2026-04-01`

## Problem

After the latest rescan, `Triolet` is the last exported form still scoring
too highly on a degenerate repeated-line poem. Reproduction:
`"\n".join(["same line again"] * 10)` returns `passed=False, score=0.870...`.
The refrain and rhyme components are all perfect, and the exact 8-line
requirement only weakly lowers the weighted score.

## Primary Files

- `src/abide/forms/triolet.py`
- `tests/fixtures/test_form_validation.py`

## Acceptance Criteria

- The repeated-line 10-line degenerate triolet must no longer score
  near-perfectly.
- Valid triolet fixtures should remain valid.
- The exported-form rescan should no longer show this outlier.

## Completed

- Added an explicit structural score-downscale in `Triolet.verify` so missing
  the exact 8-line requirement materially reduces the final score instead of
  letting perfect refrain/rhyme subscores dominate.
- Added a direct regression for the repeated-line 10-line degenerate triolet.
- Re-ran the repeated-line exported-form rescan to confirm `Triolet` is no
  longer an outlier.
