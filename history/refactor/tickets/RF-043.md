# RF-043: Stop Clerihew From Treating Bare Sentence-Start Capitalization As A Name

- Priority: `P1`
- Status: `done`
- Area: `clerihew name proxy`
- Origin: `MANUAL-VERIFIER-AUDIT-2026-04-01`

## Problem

`Clerihew._contains_name()` currently treats a lone capitalized first word as a valid name proxy. That lets ordinary lines like `Morning settled over the bay` satisfy the "first line contains a name" check and pass canonically when rhyme and line count also match.

## Primary Files

- `src/abide/forms/clerihew.py`
- `tests/fixtures/test_form_validation.py`

## Acceptance Criteria

- A bare sentence-start capitalized common word must not satisfy the clerihew name proxy.
- Existing valid clerihew fixtures with explicit names must keep passing.
- Add a direct regression for the reproduced false positive.

## Completed

- Tightened the clerihew name proxy so bare sentence-start capitalization no longer counts; it now requires a title pattern or a stronger internal or multi-token capitalized name signal.
- Added a direct regression for the reproduced non-name clerihew false positive.
- Verified with `uv run ruff check src/abide tests scripts`, `uv run mypy src/abide`, and `uv run pytest -q -W error` -> `662 passed, 1 skipped`.
