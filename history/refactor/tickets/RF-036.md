# RF-036: Reject Flattened Single-Block Pantoum Layouts Instead Of Chunking Them Back Into Quatrains

- Priority: `P1`
- Status: `done`
- Area: `pantoum stanza semantics`
- Origin: `MUTATION-SWEEP-2026-04-01`

## Problem

`Pantoum.verify()` currently passes a single-block poem with no stanza breaks because it asks `GroupedStanzas` to chunk the lines back into quatrains synthetically. That contradicts the form's own stated contract of multiple quatrain stanzas with interlocking repetition.

## Primary Files

- `src/abide/forms/pantoum.py`
- `tests/forms/`

## Acceptance Criteria

- `Pantoum` must not report `passed=True` for flattened single-block layouts.
- Keep the canonical multi-quatrain fixture passing.
- Add a direct regression for the reproduced single-block false positive.

## Completed

- Disabled synthetic single-block stanza chunking in `Pantoum` so the form now requires explicit quatrain layout.
- Added a direct regression covering the reproduced flattened single-block false positive.
- Verified with `uv run ruff check src/abide tests scripts`, `uv run mypy src/abide`, and `uv run pytest -q -W error` -> `627 passed, 1 skipped`.
