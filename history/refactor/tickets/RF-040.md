# RF-040: Reject Flattened Single-Block Blues Layouts Instead Of Chunking Them Back Into Tercets

- Priority: `P1`
- Status: `done`
- Area: `blues stanza semantics`
- Origin: `MUTATION-SWEEP-2026-04-01`

## Problem

`BluesPoem.verify()` currently passes a single-block poem with no stanza breaks because it asks `GroupedStanzas` to chunk the lines back into tercets synthetically. That contradicts the form's own stated AAB tercet structure.

## Primary Files

- `src/abide/forms/blues.py`
- `tests/forms/`

## Acceptance Criteria

- `BluesPoem` must not report `passed=True` for flattened single-block layouts.
- Keep the canonical multi-tercet fixture passing.
- Add a direct regression for the reproduced single-block false positive.

## Completed

- Disabled synthetic single-block stanza chunking in `BluesPoem` so the form now requires explicit tercet layout.
- Added the blues fixture to the shared stanza-flattening mutation harness.
- Verified with `uv run ruff check src/abide tests scripts`, `uv run mypy src/abide`, and `uv run pytest -q -W error` -> `636 passed, 1 skipped`.
