# RF-023: Harden Haiku And Tanka Exact-structure Pass Semantics In Lenient Mode

- Priority: `P1`
- Status: `done`
- Area: `Japanese short-form verifiers`
- Origin: `FOLLOWUP-015`

## Problem

`Haiku(strict=False)` and `Tanka(strict=False)` still report `passed=True` for structural near misses that should only receive partial reward. In current code, an extra trailing line or a split into multiple stanzas can leave the blended score above threshold even though the defining exact line-count and one-stanza requirements are violated.

## Primary Files

- `src/abide/forms/haiku.py`
- `tests/forms/`

## Acceptance Criteria

- `Haiku` and `Tanka` must fail canonically when the poem has extra or missing lines.
- They must fail canonically when the poem is split across multiple stanzas.
- Lenient mode may still return useful partial scores, but `passed=True` must require the defining exact structure plus a compliant syllable pattern.
- Add direct regressions for the reproduced false positives and valid baseline examples.
