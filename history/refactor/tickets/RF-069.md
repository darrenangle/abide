# RF-069: Add Property-Based Mutation Coverage For Stable Fixed-Form Fixture Families

- Priority: `P2`
- Status: `done`
- Area: `regression testing`
- Origin: `RELIABILITY-HARDENING-2026-04-02`

## Problem

The current mutation coverage is stronger than before, but it is still mostly
curated and example-driven. The next reliability layer is to generate
additional near-miss mutations automatically against stable fixture-backed form
families so the suite can keep probing for degenerate passes without hand-added
cases for every new leak.

## Primary Files

- `tests/forms/test_fixture_mutation_harness.py`
- `tests/forms/test_exact_structure_mutation_harness.py`
- `tests/fixtures/poems.py`

## Acceptance Criteria

- Add property-based or generated mutation coverage on top of the existing
  fixture-backed mutation harnesses.
- Generated mutations must preserve a passing baseline fixture and then produce
  at least one canonical failure per mutation family.
- The new harness must run in the default test suite.

## Completed

- Added `tests/forms/test_property_mutation_harness.py` with Hypothesis-backed
  generated line insertion, content deletion, and partial stanza-break
  collapse mutations across stable fixture-backed form families.
- Scoped the generated mutations to form-defining failures only, excluding
  cases like the optional terminal line in `TerzaRima` or stanza-break
  collapse in `Rondeau` where the verifier does not claim that property.
- Verified the new harness in the default suite and kept the repo green under
  `ruff`, `mypy`, and `pytest -W error`.
